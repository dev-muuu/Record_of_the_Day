## 메서드

- 특정 타입에 관련된 함수
- 다른 언어와 달리 구조체와 열거형이 메서드를 가질 수 있음

### # 인스턴스 메서드

- 특정 타입의 인스턴스에 속한 함수
- 인스턴스 내부의 프로퍼티 값을 변경하거나 특정 연산 결과를 반환하는 등 인스턴스와 관련된 기능 실행
- 함수와 달리 특정 타입 내부에 구현
- 인스턴스가 존재할 때만 사용 가능
→ 함수와 차이점
- 구조체나 열거형은 값 타입이므로 메서드 앞에 mutating 키워드를 붙여서 해당 메서드가 인스턴스 내부의 값을 변경한다는 것을 명시해야 함

```swift
class LevelClass{
    var level: Int = 0
    
    func levelUp(){
        print("Level Up!")
        level += 1
    }
}

var level1: LevelClass = LevelClass()
level1.levelUp()
print(level1.level) //'1' 출력
```

```swift
struct LevelClass2{
    
    var level:Int = 0
    
    mutating func levelUp(){    //구조체, 열거형은 메서드에서 인스턴스 내부의 값 변경시 'mutating' 키워드 필요
        print("Level Up!")
        level += 1
    }
}

var level2: LevelClass2 = LevelClass2()
level2.levelUp()
print(level2.level) //'1' 출력
```

<aside>

    💡  > 클래스와 다르게, 구조체와 열거형은 메서드에서 인스턴스 프로퍼티 값 변경 시 `mutating` 키워드 사용해야 함

</aside>

- self 프로퍼티
    - 자기 자신을 가리키는 프로퍼티
    - 인스턴스를 더 명확히 지칭하고 싶을 때 사용
    - 자바의 this와 유사
    - 값 타입 인스턴스 자체의 값 치환 가능
        
        → 클래스의 인스턴스는 참조 타입이라서 self 프로퍼티에 다른 참조 할당 불가능
        
        → 구조체, 열거형 등은 self 프로퍼티 사용해 자신 자체 치환도 가능
        
    

<aside>

    👉🏻  Swift에서는 ‘내부 선언 지역변수 → 메서드 매개변수 → 인스턴스 프로퍼티’ 순으로 변수 추적

    → 내부 선언 지역변수와 인스턴스 프로퍼티의 이름이 같으며, 내부 선언에서 인스턴스 프로퍼티를 사용하고 싶을 때, self 프로퍼티를 사용하면 인스턴스 프로퍼티로 사용가능

</aside>

```swift
class LevelClass3{

    var level: Int = 0
    
    func jumpLevel(to level: Int){
        print("Level Up!")
        self.level = level  //self 프로퍼티 사용 통해 인스턴스 프로퍼티임을 알 수 있게 함
    }
}
```

```swift
class LevelClass4{
    var level: Int = 0
    func reset(){
        self = LevelClass4() //error 발생 -> 클래스는 참조 타입이라 변경 불가능
    }
}

struct LevelClass5{
    var level: Int = 0
    
    mutating func reset(){
        self = LevelClass5()    //struct는 self 값 자체 변경 가능
    }
}
```

- 인스턴스를 함수처럼 호출하도록 하는 메서드
    - callable of user-defined nominal types 구현 목적
    - 매개변수와 반환 타입만 다르다면, 개수에 제한 없이 동일한 이름의 함수 생성 가능
    - `mutating`, `throws`, `rethrows` 키워드 사용 가능

```swift
struct Puppy{
    var name: String = "멍멍이"
    
    func callAsFunction(){
        print("멍멍")
    }
    
    func callAsFunction(destination: String){  //매개변수, 반환타입 다를 경우 여러개 생성 가능
        print("\(destination)로 달려값니다")
    }
}

var puppy: Puppy = Puppy()
puppy.callAsFunction()  //'멍멍' 출력
puppy.callAsFunction(destination: "광교") //'광교로 달려값니다' 출력
```

### # 타입 메서드

- 타입 자체에 호출이 가능한 메서드
- `static` 키워드 통해 타입 메서드 임을 나타냄
- Class의 타입 메서드
    - static 정의
        
        → 상속 후 메서드 재정의 불가능
        
    - class 정의
        
        → 상속 후 메서드 재정의 가능
        
- self 프로퍼티가 타입 그 자체를 가리킴
    - 타입 메서드 내부에서 타입 이름과 self는 같은 뜻
    - 타입 메서드에서 self 프로퍼티 사용하면, 타입 프로퍼티 및 타입 메서드 호출 가능
    
    → 인스턴스 메서드에서는 self가 인스터스 지칭
    → 타입 메서드에서는 self가 타입 지칭
